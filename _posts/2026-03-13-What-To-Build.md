# What We're Building

At Contour we're building AI Deployment teams.
We're aiming to replace deployment teams across SAAS and agent companies in midmarket and enterprise.

# Who We're Replacing
The two primary functions in a deployment team is customer success managers and implementation engineers.
Let's take Samsara as an example of a customer that we're in conversations with. They have organizations that onboard onto their platform that have users of many different functions such as the fleet managers or the training instructors. Each user need to be trained to use the platform in a specific way and this is the CSMs role. 

Now on top of that, each organization holistically needs to be setup with it's specific set of features and workflows as a subset of Samsara's huge feature ecosystem and that's the implementation engineers role.


# Where we're starting
The product that we've built today, is the hands, ears, and eye's of the CSM. It's a computer use agent that can operate on the site, speak to the user, and listen to the users input to guide them through the right workflows. The reason we're starting here is because the wealth of user data that can be captured from walking users through each platform thousands of times. We'll be able to better understand the specific bespoke usage patterns of each different type of user across every unique organization. This will help us build out the deterministic long horizon agents that are necessary for the AI implementation function that we'll be tackling in the mid term/next few months.

# How are we thinking of building AI Implementation?
This is a huge problem space that's extremely difficult. We've drawn inspiration from our friends at Decagon who have pioneered the idea of agent operating playbooks. Essentially we'll own the implementation playbooks for every type of customer across every type of business.

The way we see it we'll have 2 core types of playbooks
1. The customer workflows playbook which instructs the agent on how to handle user interactions given the user input, user data and data of similar users on the platform
2. The implementation planning playbooks which instructs the agent on how to setup very specific parts of the platform, so there might be hundreds of these. This playbook will synthesize data for the specific business it's servicing and will draw upon all other contour portfolio businesses that are in the same space to construct implementation plans that are deterministic and specific to the B2B business we're serving and each customer of theirs.

We'll also be providing tools that allow the user to sandbox these playbooks and understand exactly what action each agent is taking and why while also comparing it to previous human ran implementation workflows.
